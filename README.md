# TG_BOT — Торговый бот для Telegram

## Описание
TG_BOT — это модульный телеграм-бот для автоматизации торговых операций и анализа данных. Проект включает в себя загрузку данных, выбор моделей, стратегии торговли, визуализацию и логирование.

## Структура проекта

```
TG_BOT/
├── data/                    # Директория для хранения данных (котировки, индикаторы и т.д.)
├── logs/                   # Логи работы бота и ошибок
├── models/                 # Сохранённые ML-модели и конфигурации
├── venv/                   # Виртуальное окружение Python (не добавляется в git)
├── .env                    # Файл с переменными окружения (токены, ключи API)
├── bot_complete.py         # Основной файл запуска бота
├── config.py               # Конфигурационные параметры бота
├── data_loader.py          # Модуль загрузки и обработки данных
├── debug_bot.py            # Скрипт для отладки бота
├── logger.py               # Настройка логирования
├── model_selector.py       # Выбор и загрузка моделей
├── requirements.txt        # Зависимости проекта
├── strategy.py             # Торговые стратегии
├── visualization.py        # Модуль визуализации данных
└── скрины.docx             # Документ с скриншотами интерфейса
```

## Установка и настройка

1. Клонируйте репозиторий:
   ```bash
   git clone <repository-url>
   cd TG_BOT
   ```

2. Создайте виртуальное окружение и активируйте его:
   ```bash
   python -m venv venv
   source venv/bin/activate  # для Linux/Mac
   venv\Scripts\activate     # для Windows
   ```

3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

4. Создайте файл `.env` на основе примера и укажите в нём:
   ```
   TELEGRAM_BOT_TOKEN=ваш_токен_бота
   API_KEY=ваш_ключ_биржи
   SECRET_KEY=ваш_секретный_ключ
   ```

## Использование

### Запуск основного бота:
```bash
python bot_complete.py
```

### Запуск в режиме отладки:
```bash
python debug_bot.py
```

### Конфигурация:
- Настройки бота, такие как интервалы, лимиты, торговые пары, задаются в `config.py`
- Торговые стратегии определяются в `strategy.py`
- Модели машинного обучения подгружаются через `model_selector.py`

## Логирование
Логи сохраняются в папке `logs/` с разбивкой по датам. Формат:
```
logs/
├── bot_2025-01-28.log
├── error_2025-01-28.log
```

## Визуализация
Модуль `visualization.py` предоставляет функции для построения графиков:
- Графики цен
- Индикаторы
- Торговые сигналы

## Модели
В папке `models/` хранятся:
- Обученные модели (`.pkl`, `.h5`, `.joblib`)
- Конфигурации моделей (`.json`, `.yaml`)

## Зависимости
Основные зависимости указаны в `requirements.txt`:
- python-telegram-bot
- pandas
- numpy
- matplotlib
- scikit-learn
- tensorflow/pytorch (опционально)

## Примечания
- Папка `__pycache__` и файлы типа `~$скрины.docx` автоматически генерируются системой и не требуют ручного изменения.
- Файл `.env` должен быть добавлен в `.gitignore` для безопасности.
